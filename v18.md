# Expense Tracker V18

## Critical UI & Stability Fixes

### 1. iOS Auto-Zoom & Screen Shift Fix
- **Fixed:** Resolved the issue where clicking on input fields (like Amount) caused the page to move right or get cut in half on mobile devices.
- **Cause & Solution:** Browsers (especially iOS Safari) force an auto-zoom if form inputs have a text size smaller than 16px. Updated base CSS for all inputs and dropdowns to `font-size: 16px` to block forced zooming natively. Added `overflow-x: hidden` to root structures to prevent the app sliding sideways.

### 2. Resolved Duplicate Help Section & Spacing
- **Fixed:** Cleaned up HTML index where legacy (v13) and modern help sections were rendering simultaneously.
- **Improved:** Formatted the "How To Use" instructions to be single-line entries for cleaner readability. 

### 3. Month Deletion Button Implementation 
- **Fixed:** Discovered that previous version notes cited a "Delete Month" button, but it was not being injected into the DOM.
- **Implementation:** Integrated the trash icon properly beside the month title in the app rendering cycle. 
- **Stability:** Patched the localized Timezone bug in the `deleteMonth()` logic by standardizing the date string parse (T00:00:00) so a month can't accidentally be skipped depending on the user's local time.
