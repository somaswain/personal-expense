# Expense Tracker — Version 6

## Bug Fixes & Changes from v5

1. **Options icon fixed — inline expand instead of floating menu**
   - Root cause: the floating menu relied on a global `activeMenuId` that was cleared when `renderExpenses()` re-built the DOM. After clicking Edit, the DOM was wiped, so the edit form never populated.
   - Fix: replaced the floating menu with an **inline expandable row** directly under each expense in the table. Clicking ⋯ toggles a row showing Edit and Delete buttons — no floating positioning, no DOM-wipe race condition.
   - Edit now correctly populates the form and scrolls to top. Delete works immediately.

2. **Date field no longer overflows**
   - Added `min-width: 0` and `max-width: 100%` to all inputs inside `.form-row` grid cells.
   - Set `table-layout: fixed` on expense tables with explicit column widths so tables stay within the card boundary.
   - Removed excess webkit padding on the date picker icon.

3. **Auto-expand on save**
   - After saving an expense, `renderExpenses()` receives the target month and section (main/self).
   - The matching month card and sub-section are rendered open by default — user immediately sees the newly added row in the table without any manual tapping.

4. **"crafted by somaswain" tagline now visible**
   - Changed colour from `--text-muted` (#A8A49D — nearly invisible on the warm background) to `--text-secondary` (#6B6860) with `font-style: italic` to match the refined aesthetic of the theme.
